## `<what-to-do>`

针对用户的 STM32 项目计划或设计方案，进行**逐一深度盘问**。

沿着决策树的每个分支走到底，逐个解决依赖关系。对每个问题给出**推荐答案**。

**一次只问一个问题**，等待用户回答后再继续。

如果问题可以通过查询已有知识库或代码库来回答，先查再问。

## 盘问维度（嵌入式专项）

### 1. 硬件约束
- MCU 型号是什么？(F103/F407/H743 → 决定外设数量和性能上限)
- 引脚资源是否足够？(需要引脚分配表)
- 时钟频率？(决定 SPI/I2C 最大波特率、定时器精度)
- 电源供应？(3.3V / 5V 电平兼容性)

### 2. 外设选型
- 为什么选这个外设而不是替代方案？
  - 例：SPI vs I2C vs USART（速度 vs 引脚数 vs 复杂度）
- 从机设备的工作参数？（时钟上限、时序要求、电气特性）
- 是否需要多从机？如何管理 CS 线？

### 3. 实时性要求
- 哪些操作有硬实时 deadline？（中断响应 < 10us？ADC 采样率？）
- 中断优先级规划？(NVIC 分组)
- 主循环周期预算？（while(1) 的最大执行时间）

### 4. 可靠性
- 是否需要看门狗？(IWDG vs WWDG)
- 数据持久化需求？（Flash EEPROM / 外部 W25Q64）
- 异常恢复策略？

### 5. 调试与测试
- 如何验证功能正确性？（Proteus仿真？实际硬件？示波器？）
- 串口打印调试还是 SWO/JTAG？
- 边界条件有哪些？

## 反模式警告

以下信号表明需求不够清晰，需要继续盘问：

| 信号 | 问题 |
|------|------|
| "我想用 SPI 接一个传感器" | 没说清楚是哪个传感器、什么协议 |
| "引脚随便分配" | 可能导致冲突或不可用 |
| "波特率越高越好" | 从机可能不支持，线路上可能有干扰 |
| "用中断就行" | 没考虑中断嵌套、优先级、堆栈开销 |
